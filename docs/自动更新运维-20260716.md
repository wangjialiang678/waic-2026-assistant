## 一、数据架构：JSON 分两层，SQLite 只放用户数据

**内容数据**（活动/展商/情报）不用 MySQL/PostgreSQL——只读、整体重建、约 2MB，JSON 就是数据库：

```
raw/*.json                      源数据（官方 API 抓取、WaytoAGI、公众号/网络抽取、Tier4 自动采集）
   │
   ▼  build.py（归一化 → 去重 → 场馆归类 → 回填 → 权重/关系标注 → 分面统计）
build-output/data/*.json        全量成品（含议程 schedule / 嘉宾 guests / 全文）
   │
   ├──────────────► 私有层（只给 AI 后端读，nginx 不服务，外网拿不到）
   │                  SG: /opt/waic-api/data   南京: /home/backdeploy/apps/waic-api/data
   │
   ▼  make_light.py（剥离 schedule/guests/description_en/全文；description 截断；search_text 保留）
build-output/data-lite/*.json   精简索引层 → 公开 webroot（浏览器浏览/搜索用；详情走 /api/activity/{id}）
```

- **两层的铁律**：公开层（webroot/data、bundle.tar.gz）**绝不含 schedule/guests**。
  管线里有防泄露门禁（grep 检查，命中即拒发 + 告警），发布层面不可能再回归。
- **用户数据**（日程/兴趣/人脉对接/CMS/纠错）在 **SG 的 SQLite** `/opt/waic-api/user_state.db`，
  高频操作走浏览器 localStorage 即时读写，1.2s 防抖低频同步到服务端（LWW）。与内容数据完全分离。
- 南京 waic-api 是精简版（chat/detail/digest 只读接口，无用户数据端点）→ 镜像用户不产生数据分裂。

## 四、Tier 1 · 数据刷新（确定性，无 AI）

**位置**：南京 `~/waic-refresh/`（`ssh -p 799 -i ~/.ssh/nanjing_tencent.pem ubuntu@1.13.175.253`），配置集中 `.env`。
**流程**（`scripts/refresh_and_deploy.sh` v2）：
1. `refresh.py` 拉源（单源失败保旧数据，绝不清空）
2. **`fetch_intel.py`（Tier4）**：Tavily 搜 WAIC 新闻/公众号文章 → URL+标题双去重 → 追加 `raw/web/articles.json`（kind=coverage）。失败降级跳过，不 block。
3. `build.py` 构建全量（intel 自动并入新采集文章）
4. `validate.py` 健康门禁（数量区间/schema/骤降拦截）
5. **`make_digest.py`（Tier5）**：产 `digest-latest.json` → 双端 webroot（在"无变化跳过"**之前**，保证每天日期照常翻新）
6. 无变化跳过（`.last_deployed_sha`；推送失败删 sha 强制重推）
7. **分层发布（v2 核心）**：
   - `make_light.py` → data-lite；**防泄露门禁**（lite 含 schedule/guests → 拒发+告警）
   - bundle.tar.gz **重打为精简版**（build.py 产的全量 bundle 不出门）
   - 全量 → 本机私有层 `PRIVATE_DATA` + `systemctl restart waic-api`
   - 精简 → 本机 webroot（原子替换，备份 data.bak）
   - `PUSH_TO`：精简 → SG webroot；`PUSH_TO_PRIVATE`：全量 → SG 私有层 + 远程重启 SG api
   - `POST_DEPLOY`：精简 → 南京镜像
8. 每日快照 `snapshots/`（7 天）；飞书通知（webhook 未配则仅日志）

**.env 关键项**（南京）：`WEBROOT`（本地 staging webroot）、`PUSH_TO`、`PUSH_SSH`、`PUSH_TO_PRIVATE`、
`PRIVATE_DATA`、`POST_DEPLOY`、`POST_DEPLOY_DIGEST`、`TAVILY_API_KEY`、`FEISHU_APP_ID/SECRET`。

## 五、Tier 3 · 议程同步（飞书 → superbrain.html）

数据源/身份同 v1（飞书 base `GN7zb99RBaCqs6sMAGMcbzMKnBc`，OpenClaw app token）。
**v3 关键变更——双写冲突根治**：superbrain.html 有两个写入方（本管线写议程块 / Mac UI 侧整页部署），
旧实现各持副本互相覆盖（UI 更新会被 20 分钟后的旧副本推送回退）。现在**每轮先 curl SG 线上最新页面做基底**，
再正则替换 `SB_AGENDA` 块 → UI 更新自动吸收、议程始终叠加在最新页面上，双向 ≤20 分钟自愈。
门禁：小课堂 <8 或会客厅 <20 拒写；推送基准 = 上次成功推送 sha（失败自动补推）。

## 七、Tier 4/5 · 资讯采集与每日日报（2026-07-16 新增）

**Tier 4 采集**（`fetch_intel.py`）：每 2h 随 Tier1 跑（06:00–24:00 窗口）。Tavily 三组查询：
① WAIC 2026 新闻（1 天内）② `mp.weixin.qq.com` 域内公众号文章（2 天内，覆盖官方公众号）
③ 官网/主流媒体域内报道。过滤（标题/摘要须含 WAIC 关键词）→ 对全部 raw 已知 URL+归一化标题去重 →
追加 `raw/web/articles.json` → build 并入 `intel.json` → 前端「情报站」+ 日报可见。

**Tier 5 日报**（`make_digest.py` → `/digest-latest.json`，双端公开）：
```json
{ "date": "…", "yesterday_articles": [{title,publisher,url,summary}…≤20],
  "today_events": [{id,title,start_time,venue,tags,weight,waic_relation}…≤80 按重要度排序] }
```
- 每 2h 翻新（早 06:00 那轮即产生当天日报，08:00 前就绪）；会期外自动取最近会期日。
- **推送方式 = skill 拉取**（skill v2.2.0）：用户每天首次交互时 agent 拉 digest，按本地 interests
  过滤后播报（昨日文章 3-5 篇 + 今日个性化推荐）。个性化在客户端完成，服务端不碰用户兴趣。
- 网站侧 `/api/digest?interests=&day=` 继续提供实时个性化速报（已修跨天 bug）。
