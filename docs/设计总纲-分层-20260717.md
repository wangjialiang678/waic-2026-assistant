# WAIC 参展助手 · 设计总纲（分层）

> 2026-07-17。回答一个元问题：**这套系统的设计分几层、每层的规则放在哪、谁可以改、怎么生效。**
> 原则：能用"规则"表达的不写死成代码；能用"配置"表达的不写进规则；代码只做机制。

## 四层模型

```
L1 机制层   系统怎么运转          改=改代码+部署        （最硬，最少变）
L2 策略层   编排/阈值/清单        改=改配置+重启        （运营日常可调）
L3 行为层   agent 怎么表现        改=改 SKILL.md 发版    （LLM 解释执行，自升级分发）
L4 默认层   所有约定俗成的默认值   记录在案，散布于 L1-L3 （变更须写回本表）
```

**判断一个新需求放哪层**：每个用户都必须一样、且要可验证 → L1；运营想随时调 → L2；
需要理解语境、因人而异 → L3；只是个初始值 → L4。
（例：本次"早报优先展示我的日程"→ 内容组装进 L1（push.py 生成 sections），
"没开同步时提醒用户"→ L3 准则 2 自动覆盖，无需新代码。）

## L1 机制层（代码，两台服务器 + 管线）

**文档**：[架构图集](https://design.superbrain-ai.com/waic-2026/architecture.html) ·
[自动更新运维 v2](自动更新运维-20260716.md) · [推送系统设计](推送系统设计-20260716.md)

- 数据分层：全量(私有,AI) / 精简索引(公开,浏览器+bundle) / SQLite(用户数据)；防泄露门禁
- 5 条管线（Tier1 刷新每 2h / Tier3 议程 20min / Tier4 采集 / Tier5 日报 / Tier2 巡检）
- API：/api/chat(函数调用查库) /api/push(窗口编排) /api/state(LWW 同步) /api/social(双向匹配)
- 推送机制：客户端错峰轮询 + 服务端窗口组装 + 无状态去重(delivery_id)
- **早晚报的段落次序是机制**（我的日程→要闻→推荐，推荐去重已加入项）——保证每个用户一致

变更方式：改 `server/` `scripts/` → 部署两台 + 南京管线 → git 两仓。

## L2 策略层（配置，运营可改）

| 配置 | 位置 | 管什么 | 生效 |
|---|---|---|---|
| `push_config.json` | SG `/opt/waic-api/` | 窗口时刻/开放区间、定向规则(interest_any→topics)、**breaking 置顶重大新闻** | 重启 waic-api |
| `validate.py` 阈值 | 南京 scripts | 数量区间、骤降拦截线 | 下轮管线 |
| 标准兴趣标签集(23) | SKILL.md + chat.js + push_config | 画像/定向的公共词表——**三处必须一致** | 各自发版 |
| 人工合并清单 | build.py CURATED_* | 边会↔官方去重映射 | 下轮管线 |

## L3 行为层（SKILL.md = 给 AI 的规则书，自升级分发到所有 agent）

**核心是 4 条元规则（SKILL.md 规则 2），没写死的场景由 agent 按此推导**：
1. 用户自己的数据 > 推荐内容（展示次序；推荐不重复已加入项）
2. 机会性引导绝不打扰（前置未开启→顺带提一句；每日≤1 次；拒绝即永久停）
3. 个性化在用户端完成（服务端给池子；用户数据只经匿名码上报）
4. 每次触达都可行动（结尾给下一步）

具体场景（兴趣确认对齐标签、日程同步提醒、推送播报格式、人脉名片征得同意…）都是元规则的实例，
写进 SKILL.md 是为了示范，**新场景优先套元规则，而不是等新版说明书**。

变更方式：改 `agent-skill/SKILL.md` → bump version + 上传 `/SKILL.md` `/SKILL_VERSION` →
装机 agent 经"自升级"规则在下次会话/08·22 定时自检时收到。

## L4 默认层（默认规则清单——改动必须回写此表）

| 默认 | 值 | 所在层 |
|---|---|---|
| 轮询错峰 | 装机抽 0-20 随机分钟，持久化复用 | L3 |
| 推送窗口 | 09/12/16/21，开放区间宽松（如早报 07:30-11:29） | L2 |
| 去重 | delivery_id=MMDD-窗口，客户端记已展示 | L1 |
| 数据检查节流 | 同会话 15 分钟内不重查 VERSION | L3 |
| 引导额度 | 每日≤1 次，拒绝即永久停 | L3 |
| 联系方式 | 仅双向「感兴趣」后互露；可一键删除 | L1 |
| 采集 | 只追加不覆盖；URL+归一化标题去重；搜狗断链一律丢弃 | L1 |
| 管线降级 | 单源失败保旧数据；门禁不过不发布；快照 7 天可回滚 | L1 |
| skill 自升级 | 覆盖 my/skill_path 记录的真实路径；写入成功才记版本；下次会话生效 | L3 |
| 生命周期 | 全部定时任务 2026-07-20 后自动退场 | L1/L3 |
| 兴趣标签 | 23 个标准标签为唯一词表，自由文本须映射+确认 | L2/L3 |
