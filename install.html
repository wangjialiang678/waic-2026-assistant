Agent Skill

让你的 AI
读懂 WAIC。

这里说的"AI Agent"泛指你用的 AI 工具——OpenClaw（龙虾）、Hermes（爱马仕）、Claude Code、WorkBuddy，或任何有 shell 能力的 Agent。不用你动手装任何东西，让它自己去读、去取数据、回答你。

01

把下面这段话发给你的 AI

复制下面这段话，发给你的 AI Agent。它会自己读文档、下数据、装好——然后主动告诉你可以怎么开始问。

请按照 https://waic2026.superbrain-ai.com/INSTALL.md 里的说明，把这个 WAIC 2026 参展指南 Skill 装到我的环境里——你自己决定放在哪儿合适。装完之后告诉我可以怎么开始问你。

AI 会按服务器上的 INSTALL.md 做三件事：

  1. 下载说明书 SKILL.md 到它自己的 Skill 目录
  2. 下载数据压缩包 bundle.tar.gz 并解压到当前工作目录的 WAIC2026/ 子目录
  3. 主动给你几个典型问题，让你直接开始问
02

装好之后，你可以这样问

Skill 让你的 AI 对 WAIC 几乎全知。几个真实可用的对话——

01
今天下午世博中心有哪些跟 AI 芯片相关的论坛？
AI 按时间 + 地点 + 主题筛选，包含对论坛标题和详情的全文搜索。
02
我在西岸国际会展中心，接下来 2 小时附近有什么值得去？
AI 按场地和时间窗口筛选，告诉你就近的 3-5 场，附每场一句话介绍。
03
帮我规划 Day 2 的日程，尽量避免时间冲突，偏好大模型与 AI 基础类。
AI 筛选 → 检查时间重叠 → 给你一份日程表。
04
超脑 AI 孵化器这次在 WAIC 有哪些相关论坛？
AI 按超脑标签过滤，列出相关论坛的时间、地点、主题。
03

为什么这么简单

传统接入模式是："AI → 适配层 → API"。要为每个数据源写一个 MCP Server 或 API 网关，开发成本高。

我们的做法是："AI → 文件系统"。数据在本地，AI 用 cat / grep / 读 JSON 就能查。给 AI 一份 markdown 文档和 shell 访问，剩下交给它。
